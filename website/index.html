Shenanigans.

Shenan moves an encrypted secret from one machine to another without ever seeing what's inside. No storage. No accounts. Built on the SSH keys you already have.

What it is

Secrets that can't be harvested

∅

No storage, ever

The relay carries encrypted bytes it cannot read. There is no database, no buffer, no log. By the time a secret has been transmitted the relay has already forgotten the transaction.

⌗

Your SSH key is your identity

No new accounts. No new keypairs. Shenan reads the Ed25519 key already in your ~/.ssh and uses GitHub as a public key registry.

⊘

Blind relay

After authentication, the relay discards your GitHub username immediately. It holds only an ephemeral ID linked to your socket. No identity graph is ever constructed.

⧗

Tokens rotate every hour

Channels are derived client-side with HKDF over the current time window. A captured token expires in <60 minutes. Nothing useful to harvest.

Usage

Get started in seconds

Both parties must be online simultaneously — shenan is a pipe, not a mailbox.

quick start 
# discover your local SSH key, set up trust store
$ shenan init

# trust a colleague's GitHub account
$ shenan trust add github:alice

# Alice sends a secret to Bob (blocks until Bob connects)
alice $ shenan send --to github:bob DATABASE_URL=postgres://prod/mydb

# Bob receives it (blocks until Alice connects)
bob $ shenan receive --from github:alice

# write directly to a file
bob $ shenan receive --from github:alice --out .env
send multiple secrets / from a file 
# send multiple env vars inline
$ shenan send --to github:alice \
  DATABASE_URL=postgres://... \
  REDIS_URL=redis://... \
  API_KEY=sk-abc123

# pipe in a file
$ shenan send --to github:alice < .env.production
Security

Security properties

What the relay can and cannot know.

Property Guarantee
Encryption ✦End-to-end, client-side only (X25519 + ChaCha20-Poly1305)
Forward secrecy ✦Fresh ephemeral keypair per send
Relay knowledge ✦Anonymous ephemeral IDs only — username discarded after auth
Relationship graph ✦Never constructed, never stored
Ciphertext at rest ✦Structurally impossible — relay has no persistence layer
Harvest now / decrypt later ✦No surface to harvest
Relay compromise impact ✦Zero — nothing to learn, nothing to steal
